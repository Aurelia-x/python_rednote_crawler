# 小红书自动化内容生产与发布系统

## 项目简介

本项目实现了一个全流程的小红书内容自动化系统，包含从**数据采集**（爬虫）到**数据筛选**，再到**AI辅助创作**，最后**自动发布**的完整闭环。

**核心功能**：
1.  **采集端**：基于 Playwright + API 的混合模式爬虫，高效抓取指定关键词的笔记图文。
2.  **管理端**：提供数据迁移工具，从采集库中筛选优质素材至发布库。
3.  **发布端**：集成阿里云 Qwen-VL 多模态大模型进行二创，并通过浏览器自动化模拟真人行为完成笔记发布。

## 核心模块

### 1. 数据采集 (`xhs_crawler.py`)
- **混合模式**：利用 Playwright 进行 API 签名（X-s, X-t），使用 `httpx` 直接请求接口，兼顾速度与反爬能力。
- **可视化演示**：支持“演示模式”，在后台高速爬取的同时，前台浏览器同步展示正在抓取的页面。
- **数据清洗**：内置关键词过滤、黑名单机制、图片分辨率筛选等多维度清洗规则。

### 2. 数据处理 (`add_to_final.py`)
- **数据集分离**：将爬取的原始数据 (`data/`) 与待发布数据 (`data_final/`) 物理隔离，确保发布内容的质量。
- **人工审核**：通过配置列表选择特定笔记，将其从原始库迁移到最终发布库。
- **额外说明**：由于严格的六宫格主题漫画较少，最终的数据保留了相关主题的手绘多图/多格漫画，并不严格按照六宫格实行。已在讲解视频中说明。

### 3. 自动发布 (`xhs_publisher.py`)
- **AI 赋能**：集成 Qwen-VL 大模型，根据图片内容自动生成符合小红书风格的“爆款”标题和正文。
- **模拟真人**：完全基于 Playwright 的浏览器自动化操作，自动上传图片、填写文案、处理弹窗、规避风控。
- **干跑模式**：支持 Dry Run 模式，仅执行操作流程但不点击最终发布按钮，便于测试。

## 快速开始

### 环境准备

1.  **Python 环境**：确保安装 Python 3.8+。
2.  **安装依赖**：
    ```bash
    pip install -r requirements.txt
    playwright install chromium
    ```
3.  **配置文件**：
    - 在项目根目录创建 `.env` 文件，配置阿里云 DashScope API Key（用于 AI 文案生成）：
      ```
      DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxx
      ```

### 使用流程

#### 第一步：获取登录凭证
运行 `get_cookies.py`，手动扫码登录小红书账号。脚本会自动保存 Cookie 到 `cookies.json`。
```bash
python get_cookies.py
```

#### 第二步：抓取素材
运行爬虫脚本，抓取指定关键词的笔记。
```bash
python xhs_crawler.py
```
*注：可在代码中修改 `KEYWORDS` 列表调整搜索词。*

#### 第三步：筛选素材
1.  打开 `data/image` 目录，浏览并记录想要发布的笔记 ID（文件夹名称）。
2.  打开 `add_to_final.py`，修改 `TARGET_NOTE_IDS` 列表，填入选中的 ID。
3.  运行脚本：
    ```bash
    python add_to_final.py
    ```

#### 第四步：自动发布
运行发布脚本，自动读取 `data_final/` 中的素材并发布。
```bash
python xhs_publisher.py
```
*注：发布脚本默认开启 `DRY_RUN = True`（不实际点击发布），确认无误后可在代码中将其改为 `False`。*

## 注意事项

- **合规性**：本项目仅供学习与研究使用，请勿用于大规模商业抓取或发布垃圾信息，遵守小红书平台规则。
- **账号安全**：建议使用测试账号进行实验，高频操作可能导致账号被风控。
